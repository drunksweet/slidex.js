## 1. 功能概述

tang-slidex 的 **Edit Mode** 允许用户在浏览器内直接编辑当前幻灯片，无需打开代码编辑器：

| 能力 | 描述 |
|------|------|
| **文本内联编辑** | 双击任意文字区域即可修改，`contenteditable` 驱动 |
| **元素拖拽移动** | 拖动任意块级元素改变位置，实时预览 |
| **对齐辅助线** | 拖拽时接近水平/垂直中轴自动出现橙色辅助线 |
| **选中框** | 点击元素显示蓝色选中框 + 元素类型标签 |
| **删除元素** | 选中后点击 ✕ 按钮删除 |
| **取消还原** | `Esc` 或取消按钮一键撤销所有本次编辑 |
| **保存回写** | `Ctrl+S` 将变更写回 `.html` 源文件，Vite HMR 自动刷新 |

**零侵入设计**：所有编辑能力均在 `index.html` 中实现，**不修改任何 slide HTML 文件的结构**。只有点击"保存"时才通过 Vite dev server 中间件将 patch 写回文件。

## 3. 核心模块：EditManager

位于 `examples/tech-intro/index.html` 的 `<script type="module">` 块中，以 ES2022 私有类实例化：

```javascript
const editManager = new class EditManager {
  enable()   // 进入编辑模式，扫描 DOM 注入 contenteditable + 拖拽事件
  disable()  // 退出（不保存），还原所有文字和 transform
  save()     // POST patches 到 /api/save-slide，成功后退出
  bind()     // 扫描当前 slide，注入可编辑能力
  updateOverlay(el)  // 更新蓝色选中框位置
}
```

### 3.1 文本编辑

`_collectTextElements(root)` 通过 `TreeWalker` 遍历 slide DOM，筛选出**只含文本的叶子节点**（跳过 `canvas`、`svg`、`script`、`style`），对每个叶子节点：

1. 保存原始 `textContent` 快照到 `_snapshots: Map`
2. 设置 `contentEditable = 'true'`，添加 `.tang-editable` 样式类
3. 监听 `blur` 事件：若内容变更，记录 `{ type: 'text', selector, value }` patch

### 3.2 元素拖拽

对 `.slide` 的所有直接子元素（排除 `style`、`canvas`、`script`）注入 `mousedown` 监听：

```
mousedown → 记录起始坐标 (startX, startY)
mousemove → el.style.transform = translate(baseX + dx/scale, baseY + dy/scale)
mouseup   → 记录 { type: 'move', selector, dx, dy } patch
```

**关键细节**：鼠标坐标需除以当前 `scale`（`scaleToFit()` 的缩放比例），还原为 slide 内部坐标。

### 3.3 Selector 生成

`_getSelector(el)` 从目标元素向上回溯至 `#slide-host`，拼接 `:nth-child(N)` 路径：

```javascript
// 示例输出：
'#slide-host > :nth-child(1) > :nth-child(3) > :nth-child(2)'
```

该路径与 class/id 无关，对 AI 生成的任意 slide HTML 都稳定可用。服务端 JSDOM 用同一 selector 定位元素并应用变更。

### 3.4 变更追踪

`patches: PatchOp[]` 数组存储本次会话的所有变更。同一 selector 的重复操作会去重，保留最新值：

```javascript
// 文本变更去重示例
this.patches = this.patches.filter(p => !(p.type === 'text' && p.selector === selector))
this.patches.push({ type: 'text', selector, value: current })
```

## 4. Vite 插件：slideSavePlugin

位于 `examples/tech-intro/vite.config.js`，在 dev server 上注册 `POST /api/save-slide` 中间件：

```javascript
function slideSavePlugin() {
  return {
    name: 'tang-slidex-save',
    configureServer(server) {
      server.middlewares.use('/api/save-slide', (req, res) => {
        // 1. 读取请求体 { slideIndex, patches }
        // 2. 定位 slides/slide-NNN.html
        // 3. JSDOM 解析 → applyPatches() → fs.writeFileSync
        // 4. 返回 { ok: true }
        // Vite 的 fs watcher 自动感知文件变化 → HMR tang-slide-update
      })
    }
  }
}
```

### applyPatches()

```javascript
function applyPatches(html, patches) {
  const dom = new JSDOM(html)
  for (const patch of patches) {
    const el = dom.window.document.querySelector(patch.selector)
    switch (patch.type) {
      case 'text':   el.textContent = patch.value; break
      case 'move':   // 累加 data-edit-dx/dy，注入 transform: translate()
      case 'resize': // 注入 width/height style
      case 'delete': el.parentNode.removeChild(el); break
    }
  }
  return dom.serialize()
}
```

JSDOM 依赖：`pnpm add -D jsdom`（已添加到 `examples/tech-intro/package.json`）。

## 5. Patch 数据模型

```typescript
type PatchOp =
  | { type: 'text';   selector: string; value: string }
  | { type: 'move';   selector: string; dx: number; dy: number }
  | { type: 'resize'; selector: string; width: number; height: number }
  | { type: 'delete'; selector: string }

interface SaveRequest {
  slideIndex: number   // 0-based，与文件名 slide-NNN.html 的 NNN-1 对应
  patches: PatchOp[]
}
```

**selector 格式**：`#slide-host > :nth-child(N) > …`（nth-child 路径，不依赖 class/id）

## 6. UX 交互说明

### 6.1 进入/退出编辑模式

| 操作 | 快捷键 / 按钮 |
|------|--------------|
| 进入编辑模式 | 导航栏 **✏️ 编辑** 按钮 / 键盘 `E` |
| 保存并退出 | 导航栏 **💾 保存** 按钮 / `Ctrl+S` |
| 取消（撤销所有变更）| 导航栏 **✕ 取消** 按钮 / `Esc` |

激活后的视觉反馈：
- 顶部出现**橙色提示条**（显示快捷键提示）
- `#slide-host` 显示橙色外边框
- 导航栏切换为橙色调

### 6.2 文本编辑

- 进入编辑模式后，所有叶子文本元素自动加**蓝色虚线轮廓**
- 光标移入变为文字输入样式
- 点击聚焦后变为**实线蓝框**
- 失焦时自动记录变更

### 6.3 拖拽移动

- 块级元素（slide 直接子元素）变为 `cursor: grab`
- 拖拽时变为 `cursor: grabbing`，透明度降低
- 接近 slide 水平/垂直中轴时出现**橙色对齐辅助线**（8px 吸附阈值）

### 6.4 选中框

- 点击任意块级元素 → 出现**蓝色虚线边框** + 左上角**元素标签**（如 `div.card`）
- 右上角 **✕ 按钮** → 删除该元素（前端隐藏 + 记录 delete patch）
- 随拖拽实时更新位置

### 6.5 保存流程

```
用户点击"保存"
  → EditManager.save()
  → POST /api/save-slide { slideIndex, patches }
  → slideSavePlugin 中间件
  → JSDOM 解析 + applyPatches
  → fs.writeFileSync → 文件变更
  → Vite HMR watcher → tang-slide-update 事件
  → 前端 navigateTo(current, { hmrReload: true })
  → toast "💾 已保存"
```

## 7. 约束与边界

| 场景 | 处理方式 |
|------|---------|
| **ECharts canvas 图表** | `_collectTextElements` 跳过 `canvas`；拖拽也不绑定含 canvas 的子树 |
| **Canvas 粒子画布** | 同上，跳过 |
| **`overflow: hidden` 容器** | 拖拽超出 slide 边界后视觉被裁剪，未做强制 clamp（可后续加） |
| **缩放 `scale(x)` 影响坐标** | `mousemove` 中坐标差值除以 `currentScale` 还原 |
| **生产模式无 Vite** | 仅在 `import.meta.hot` 存在时显示编辑按钮（`/api/save-slide` 不存在） |
| **编辑模式下翻页** | 禁用翻页按钮和键盘方向键，防止误操作丢失变更 |
| **HMR 与编辑态冲突** | `hmrReload: true` 时允许重新渲染，渲染后重新调用 `editManager.bind()` |

## 8. 后续迭代方向

### 近期（Phase 2）

- **Resize 手柄**：选中框四角加拖拽 resize，写入 `width/height` patch
- **右键菜单**：复制、粘贴、层级调整（z-index）
- **撤销/重做栈**：`Ctrl+Z` / `Ctrl+Shift+Z`，基于 patch 反向操作

### 中期（Phase 3 — Electron Editor）

- 将 EditManager 迁移为 `@tang-slidex/editor` 包的核心模块
- 配合三栏 IDE 布局：左侧文件树、中间预览、右侧属性面板
- 属性面板实时同步选中元素的 CSS 变量、字号、颜色
- IPC 通信代替 `fetch /api/save-slide`

### 长期

- **AI 辅助编辑**：选中元素后，侧边栏输入自然语言指令（"把这段文字改成更简洁"），Agent 自动生成新文案
- **版本历史**：基于 git diff 的 slide 历史回滚
- **协同编辑**：WebSocket + OT/CRDT patch 合并
