## 1. 核心问题

tang-slidex 的每个 slide 是一个**纯 HTML 文件**。编辑系统需要打通两个方向：

```
正向（用户操作 → 代码）
  浏览器可视化操作 ──→ 映射为源码修改 ──→ 写回 .html 文件

逆向（代码 → 用户视角）
  Agent / 开发者修改 .html ──→ Vite HMR ──→ 浏览器实时刷新
```

逆向方向（HMR）已有完整实现。本文档聚焦**正向方向**的技术方案。

### 1.1 现有方案的根本局限

现有 `EditManager` 使用 `nth-child` selector + JSDOM 序列化：

```
浏览器操作 → patches[{ selector, value }] → POST /api/save-slide
          → JSDOM 解析 → querySelector → 修改 DOM → serialize()
```

| 缺陷 | 说明 |
|------|------|
| **Selector 脆弱** | `#slide-host :nth-child(3) > :nth-child(2)` 稍改 HTML 结构即失效 |
| **格式全损** | JSDOM 序列化重排所有 HTML，注释丢失、缩进变化 |
| **无行号概念** | 无法精确定位元素对应源文件的哪一行 |
| **AI 无上下文** | 没有精确位置信息，无法为 Agent 提供可用的编辑锚点 |

### 2.2 三层分类：技术实现路径

所有操作按"对 HTML 源码的修改复杂度"分为三层：

```
┌─────────────────────────────────────────────────────────────────┐
│  第一层：改属性值                                                 │
│  → 行号 + 字符串替换（确定性操作，格式零损耗）                    │
│  → 适合：文字修改、位置/尺寸、颜色、透明度、圆角等               │
├─────────────────────────────────────────────────────────────────┤
│  第二层：改代码块                                                 │
│  → 需要 AST 级别操作或 AI                                        │
│  → 适合：<style> 规则修改、ECharts JS 配置、动画 GSAP 代码       │
├─────────────────────────────────────────────────────────────────┤
│  第三层：生成新内容                                               │
│  → AI 天然适合（创造性任务，无确定性写法）                        │
│  → 适合：新增元素、新增页面、复杂重排版、图表类型切换             │
└─────────────────────────────────────────────────────────────────┘
```

| 层级 | 操作类型 | 实现方式 | 主要工具 |
|------|---------|---------|---------|
| **第一层** | 文字、位置、尺寸、颜色、透明度、圆角、边框、删除 | 精确行号 + 字符串替换 | Vite 插件 + 行级 patch |
| **第二层** | `<style>` 规则、ECharts option、GSAP 动画代码 | 元素级 HTML 替换 | AI Agent（有上下文约束）|
| **第三层** | 新增元素、新增页面、整页重排版、图表类型切换 | AI 生成完整 HTML | AI Agent（自由生成）|

**关键洞察**：路线一（WYSIWYG）覆盖**第一层全部**操作，配合工具面板可非常完整。路线二（AI 辅助）覆盖**第二层和第三层**。这两层的边界清晰，不存在"谁能力不足"的问题，而是各自在合适的场景里发挥作用。

## 4. 共享基础设施：行号注入机制

两种路线共享同一套"元素定位"基础设施，借鉴 [code-inspector](https://github.com/zh-lx/code-inspector) 的构建时注入原理。

### 4.1 Vite transform 插件（slideInspectorPlugin）

```javascript
// vite.config.js

function slideInspectorPlugin() {
  return {
    name: 'tang-slidex-inspector',
    // 仅在开发模式，拦截 slides/*.html 的加载
    transform(code, id) {
      if (!id.match(/slides\/slide-\d+\.html$/)) return null
      if (process.env.NODE_ENV !== 'development') return null
      return injectLineNumbers(code, id)
    }
  }
}

function injectLineNumbers(html, filePath) {
  const lines = html.split('\n')
  // 跳过不可编辑标签
  const SKIP_TAGS = new Set(['style','script','meta','link','head','html','body'])
  
  const result = lines.map((line, i) => {
    return line.replace(
      /<([a-zA-Z][a-zA-Z0-9]*)(\s[^>]*)?(\/?>)/g,
      (match, tag, attrs = '', close) => {
        if (SKIP_TAGS.has(tag.toLowerCase())) return match
        if (close === '/>') return match  // void element，跳过
        const rel = path.relative(process.cwd(), filePath)
        return `<${tag}${attrs} data-tang-line="${i + 1}" data-tang-file="${rel}"${close}`
      }
    )
  })
  return result.join('\n')
}
```

**注入效果（仅内存，不修改源文件）：**

```html
<!-- 原始 slide-001.html 第 10 行 -->
<h1 class="slide-title">tang-slidex</h1>

<!-- Vite transform 后 -->
<h1 class="slide-title" data-tang-line="10" data-tang-file="slides/slide-001.html">tang-slidex</h1>
```

### 4.2 通用锚点获取

浏览器端统一使用 `_getAnchor(el)` 替代现有的 `_getSelector(el)`：

```javascript
function getAnchor(el) {
  const line = el.dataset.tangLine
  const file = el.dataset.tangFile
  if (line && file) {
    return { type: 'line', file, line: parseInt(line) }
  }
  // fallback（极少情况）
  return { type: 'selector', selector: getNthChildSelector(el) }
}
```

### 4.3 写回时清理

服务端写回文件前，移除 `data-tang-*` 注入属性，保持源文件干净：

```javascript
function cleanInjectAttrs(html) {
  return html
    .replace(/\s+data-tang-line="\d+"/g, '')
    .replace(/\s+data-tang-file="[^"]*"/g, '')
}
```

## 5. 路线一：WYSIWYG 可视化编辑（属性值操作层）

覆盖**第一层**所有操作，即：**改 HTML 属性和文本内容**（确定性操作）。

### 5.1 Patch 数据结构

```typescript
// 所有 WYSIWYG 操作最终转化为 Patch[]

type LineAnchor = { type: 'line'; file: string; line: number }

type Patch =
  | { type: 'text';        anchor: LineAnchor; value: string; original: string }
  | { type: 'move';        anchor: LineAnchor; dx: number; dy: number }
  | { type: 'resize';      anchor: LineAnchor; width: number; height: number }
  | { type: 'style-prop';  anchor: LineAnchor; property: string; value: string }
  | { type: 'class-add';   anchor: LineAnchor; className: string }
  | { type: 'class-remove';anchor: LineAnchor; className: string }
  | { type: 'attr-set';    anchor: LineAnchor; attr: string; value: string }
  | { type: 'delete';      anchor: LineAnchor }
```

**各操作对应的 Patch 类型：**

| 用户操作 | Patch 类型 | 写回方式 |
|---------|-----------|---------|
| 双击修改文字 | `text` | 替换标签内文本 |
| 拖拽移动 | `move` | 修改/新增 `style="transform:translate(...)"` |
| 拖拽 resize | `resize` | 修改/新增 `style="width:Xpx;height:Ypx"` |
| 改字色/背景色 | `style-prop` | 修改内联 style 中的单个属性 |
| 改透明度 | `style-prop` | `opacity: 0.8` |
| 改圆角 | `style-prop` | `border-radius: 12px` |
| 添加动画类 | `class-add` | 在 `class="..."` 中追加类名 |
| 删除动画 | `class-remove` | 从 `class="..."` 中移除类名 |
| 修改图片 src | `attr-set` | `src="..."` 属性替换 |
| 删除元素 | `delete` | 注释掉整个元素块 |

### 5.2 服务端行级写回（废弃 JSDOM）

```javascript
function applyPatchesByLine(html, patches) {
  const lines = html.split('\n')

  for (const patch of patches) {
    if (patch.anchor.type !== 'line') continue
    const i = patch.anchor.line - 1
    if (!lines[i]) continue

    switch (patch.type) {
      case 'text':
        // 只替换 >内容</tag> 中的内容，保留标签及所有属性
        lines[i] = lines[i].replace(/(>[^<]*)(<\/)/, `>${escapeHtml(patch.value)}$2`)
        break

      case 'move': {
        const newTransform = `translate(${patch.dx}px, ${patch.dy}px)`
        lines[i] = setInlineStyleProp(lines[i], 'transform', newTransform)
        break
      }

      case 'resize':
        lines[i] = setInlineStyleProp(
          setInlineStyleProp(lines[i], 'width', `${patch.width}px`),
          'height', `${patch.height}px`
        )
        break

      case 'style-prop':
        lines[i] = setInlineStyleProp(lines[i], patch.property, patch.value)
        break

      case 'class-add':
        lines[i] = lines[i].replace(
          /class="([^"]*)"/,
          (_, cls) => `class="${cls.trim()} ${patch.className}".trim()"`
        )
        break

      case 'class-remove':
        lines[i] = lines[i].replace(
          /class="([^"]*)"/,
          (_, cls) => `class="${cls.split(' ').filter(c => c !== patch.className).join(' ')}"`
        )
        break

      case 'attr-set':
        // 如果属性已存在则替换，否则新增
        if (lines[i].includes(`${patch.attr}="`)) {
          lines[i] = lines[i].replace(new RegExp(`${patch.attr}="[^"]*"`), `${patch.attr}="${patch.value}"`)
        } else {
          lines[i] = lines[i].replace(/>/, ` ${patch.attr}="${patch.value}">`)
        }
        break

      case 'delete':
        // 找到闭合标签，整块注释掉
        const end = findClosingLine(lines, i)
        const block = lines.slice(i, end + 1).join('\n')
        lines.splice(i, end - i + 1, `<!-- [tang-deleted]\n${block}\n-->`)
        break
    }
  }

  return cleanInjectAttrs(lines.join('\n'))
}

// 工具函数：修改内联 style 中的单个属性
function setInlineStyleProp(line, prop, value) {
  const styleMatch = line.match(/style="([^"]*)"/)
  if (styleMatch) {
    const existing = styleMatch[1]
    // 如果已有该属性，替换；否则追加
    const updated = existing.match(new RegExp(`${prop}\\s*:`))
      ? existing.replace(new RegExp(`${prop}\\s*:[^;]*`), `${prop}: ${value}`)
      : `${existing.trimEnd()}; ${prop}: ${value}`
    return line.replace(/style="[^"]*"/, `style="${updated}"`)
  }
  // 没有 style 属性，新增
  return line.replace(/(\/?>)/, ` style="${prop}: ${value}"$1`)
}

// 工具函数：找到第 i 行开标签对应的闭合行（标签栈）
function findClosingLine(lines, startIdx) {
  const tagMatch = lines[startIdx].match(/<([a-zA-Z][a-zA-Z0-9]*)/)
  if (!tagMatch) return startIdx
  const tag = tagMatch[1]
  let depth = 0
  for (let i = startIdx; i < lines.length; i++) {
    depth += (lines[i].match(new RegExp(`<${tag}[\\s>]`, 'g')) || []).length
    depth -= (lines[i].match(new RegExp(`</${tag}>`, 'g')) || []).length
    if (depth === 0) return i
  }
  return startIdx
}
```

### 5.3 EditManager 升级要点

现有 `EditManager` 需要以下升级：

1. **`_getAnchor(el)`** 替代 `_getSelector(el)`：读取 `data-tang-line`
2. **Patch 结构升级**：所有操作使用新的 Patch 类型定义
3. **工具面板**：每个操作（颜色、字号、对齐等）通过工具面板触发，生成对应 Patch
4. **多选支持**：Shift 点击 / 框选生成多个 anchor，批量处理 Patch


## 6. 路线二：AI Agent 辅助编辑（代码块 / 生成层）

覆盖**第二层**（改代码块）和**第三层**（生成新内容）。

### 6.1 核心思路：选区 = 精确上下文信号

用户选中 slide 上的某个区域，本质是在告诉 Agent：

> "我接下来的指令，是针对这个区域/元素的，你有精确的行号锚点"

行号信息让 Agent 能够做到：
- 知道要修改的是哪个元素（第几行）
- 修改后精确替换回源文件（不影响其他行）

### 6.2 操作分层路由

```
用户发出 AI 指令
  │
  ├─ 第二层（改代码块）
  │   例："把这个图表改成折线图"
  │   → Agent 收到：图表元素的行号 + 当前 <script> 中的 ECharts option 代码
  │   → Agent 返回：replace-element（整个图表元素替换）
  │   → 回写方式：replaceElementAtLine(html, line, newHtml)
  │
  └─ 第三层（生成新内容）
      例："在标题下方加一个带三个要点的列表"
      → Agent 收到：标题元素行号 + 完整 slide HTML
      → Agent 返回：insert-after（在指定行后插入新 HTML）
      → 回写方式：lines.splice(line, 0, newHtml)
```

### 6.3 SelectionManager

```javascript
class SelectionManager {
  constructor() {
    this.selectedElements = []
    this.mode = 'none'  // 'none' | 'element' | 'range'
  }

  // 单元素选中（单击块级元素）
  selectElement(el) {
    this.selectedElements = [el]
    this.mode = 'element'
    this._highlight()
    this._emit()
  }

  // 范围框选（拖拽 → 矩形区域内所有元素）
  selectByRect(rect) {
    const slide = document.querySelector('.slide')
    this.selectedElements = Array.from(slide.querySelectorAll('*')).filter(el => {
      if (['STYLE','SCRIPT','CANVAS'].includes(el.tagName)) return false
      const r = el.getBoundingClientRect()
      return !(r.right < rect.left || r.left > rect.right ||
               r.bottom < rect.top || r.top > rect.bottom)
    })
    this.mode = 'range'
    this._highlight()
    this._emit()
  }

  // 序列化为 Agent 可用的上下文
  toAgentContext(currentSlide) {
    return {
      slideFile: `slides/slide-${String(currentSlide + 1).padStart(3,'0')}.html`,
      slideIndex: currentSlide,
      selectionMode: this.mode,
      // 选中元素的精确信息（含行号）
      fragments: this.selectedElements.map(el => ({
        tagName: el.tagName.toLowerCase(),
        classList: [...el.classList].join(' '),
        line: el.dataset.tangLine ? parseInt(el.dataset.tangLine) : null,
        file: el.dataset.tangFile ?? null,
        html: cleanInjectAttrs(el.outerHTML),
        textContent: el.textContent.trim()
      })),
      // 完整 slide HTML（宽上下文）
      fullSlideHtml: cleanInjectAttrs(document.querySelector('.slide')?.outerHTML ?? '')
    }
  }

  clear() {
    this.selectedElements = []
    this.mode = 'none'
    this._clearHighlight()
```

```javascript
async function applyAgentEdit(response, slideFile) {
  const { editType } = response

  switch (editType) {
    case 'patch': {
      // 第一层兼容：复用 applyPatchesByLine
      let html = fs.readFileSync(slideFile, 'utf8')
      html = applyPatchesByLine(html, response.patches)
      saveWithHistory(slideFile, html)
      break
    }

    case 'replace-element': {
      // 第二层：按行号替换整个元素块
      let html = fs.readFileSync(slideFile, 'utf8')
      for (const r of response.elementReplacements) {
        html = replaceElementAtLine(html, r.line, r.newHtml)
      }
      saveWithHistory(slideFile, html)
      break
    }

    case 'insert-after': {
      // 第三层：在指定行后插入新 HTML
      let html = fs.readFileSync(slideFile, 'utf8')
      const lines = html.split('\n')
      // 逆序处理，避免行号偏移
      for (const ins of [...response.insertions].reverse()) {
        lines.splice(ins.line, 0, ins.newHtml)
      }
      saveWithHistory(slideFile, lines.join('\n'))
      break
    }

    case 'replace-slide': {
      // 第三层：整页重写
      saveWithHistory(slideFile, response.newSlideHtml)
      break
    }
  }
}

// 按行号替换整个元素（标签栈追踪）
function replaceElementAtLine(html, line, newHtml) {
  const lines = html.split('\n')
  const endLine = findClosingLine(lines, line - 1)
  lines.splice(line - 1, endLine - line + 2, newHtml)
  return lines.join('\n')
}
```

### 7.3 Undo/Redo

```javascript
// 服务端内存历史栈（简单方案，无需 git）
const editHistory = new Map()  // filepath → htmlSnapshot[]

function saveWithHistory(filePath, newHtml) {
  if (!editHistory.has(filePath)) editHistory.set(filePath, [])
  const history = editHistory.get(filePath)
  history.push(fs.readFileSync(filePath, 'utf8'))  // 保存当前版本快照
  if (history.length > 30) history.shift()
  fs.writeFileSync(filePath, newHtml, 'utf8')
}

// POST /api/undo
server.middlewares.use('/api/undo', (req, res) => {
  const { file } = parseBody(req)
  const history = editHistory.get(file)
  if (!history?.length) return res.json({ ok: false, message: '无更多历史' })
  fs.writeFileSync(path.join(process.cwd(), file), history.pop(), 'utf8')
  res.json({ ok: true })
})
```

客户端 `Ctrl+Z` → POST `/api/undo` → Vite HMR 刷新。

## 9. 技术选型

### 9.1 HTML 解析（行号注入）

| 方案 | 优点 | 缺点 | 适用阶段 |
|------|------|------|---------|
| 正则逐行替换 | 零依赖，快速 | 复杂属性边界情况不稳定 | MVP |
| `htmlparser2` | 轻量（6KB），事件驱动，保留格式 | 需自行重建 HTML 字符串 | 稳定版 |
| `node-html-parser` | API 友好 | 序列化时有格式变化风险 | 备选 |

**推荐**：MVP 用正则，稳定后迁移到 `htmlparser2`。
