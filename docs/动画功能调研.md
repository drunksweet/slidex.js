## 1. 两个动画层次

tang-slidex 的动画需求分为两个正交的层次：

```
┌─────────────────────────────────────────────────┐
│  层次一：页面级切换动画（Page Transition）         │
│  发生时机：从 slide N → slide N+1 的瞬间           │
│  典型效果：淡入淡出、左右滑动、缩放、翻页           │
└─────────────────────────────────────────────────┘
         ▼ 切换完成后，下一页开始

┌─────────────────────────────────────────────────┐
│  层次二：元素入场动画（Build / Element Animation） │
│  发生时机：用户在同一页内按空格/箭头/点击            │
│  核心概念：一页内有 N 个"步骤"，逐步 reveal 元素   │
│  典型效果：淡入、从下飞入、弹出、打字机效果          │
│  全部 reveal 完 → 再按一次 → 才触发页面切换         │
└─────────────────────────────────────────────────┘
```

## 2. 层次一：页面切换动画（Page Transition）

### 2.1 交互触发时机

| 触发方式 | 现状 | 期望 |
|---------|------|------|
| 空格 / → | 立即切页 | 先执行切换动画再切页 |
| 点击空白区域 | 立即切页 | 先执行切换动画 |
| 触摸滑动 | 立即切页 | 先执行滑动动画 |

### 2.2 内置切换效果清单

| 效果名 | 描述 | 优先级 |
|--------|------|--------|
| `fade` | 淡入淡出（当前 fade out + 下一页 fade in） | P0，必做 |
| `slide-left` | 向左滑出（Next），向右滑出（Prev） | P0，必做 |
| `slide-up` | 向上推出 | P1 |
| `zoom` | 当前页缩小消失，下一页放大进入 | P1 |
| `none` | 无动画瞬切 | P0，必做 |

### 2.3 切换动画的作用对象

```
#slide-host 容器（1280×720 的 DOM 节点）
  ├── 当前 slide DOM（outgoing）
  └── 下一页 slide DOM（incoming）
```

两页 DOM 同时存在于 `#slide-host`（或其外层）才能实现真正的交叉动画。
当前 `useSlideRunner` 是"清空 innerHTML 再注入"，需要稍作改造：
**在动画期间临时保留旧 slide DOM，动画完成后再清除**。

### 2.4 配置项（Slide 级）

每页 slide 可在 HTML 中声明自己希望的入场切换效果：

```html
<!-- 方式 A：data 属性（推荐，Agent 易生成） -->
<div class="slide"
     data-slide-id="003"
     data-slide-type="content"
     data-transition-in="fade"
     data-transition-out="slide-left">
```

或者在全局 `tang-slidex.config.json` 里配置默认值：

```json
{
  "totalSlides": 16,
  "defaultTransition": "fade"
}
```

## 3. 层次二：元素入场动画（Build Animation）

### 3.1 核心概念：步骤（Step）

一页 slide 内，元素被分配到若干"步骤"。
- **步骤 0**（`step="0"`）：页面加载后立即显示的元素（基础骨架）
- **步骤 1**（`step="1"`）：第一次"点击推进"时显示的元素
- **步骤 2**（`step="2"`）：第二次"点击推进"时显示的元素
- ...依此类推
- **步骤 last → 切页**：所有步骤都展示完后，下一次点击触发页面切换

```html
<div class="slide" data-total-steps="3">
  <!-- step="0"：页面进入时立即可见（无需点击） -->
  <h1 class="slide-title" data-step="0">标题</h1>

  <!-- step="1"：第1次点击后 reveal -->
  <div class="point" data-step="1" data-animation="fade-up">要点一</div>

  <!-- step="2"：第2次点击后 reveal -->
  <div class="point" data-step="2" data-animation="fade-up">要点二</div>

  <!-- step="3"：第3次点击后 reveal；全部完成后再点一次才切页 -->
  <div class="summary" data-step="3" data-animation="zoom-in">总结</div>
</div>
```

### 3.2 步骤推进规则

```
打开页面
  → 所有 data-step="0" 的元素立即显示
  → 所有 data-step≥1 的元素初始 hidden（opacity:0 + visibility:hidden 或 display:none）

用户点击 / 按空格
  → 当前步骤 < totalSteps：reveal 下一批同一 step 的元素（并行播放动画）
  → 当前步骤 === totalSteps：触发页面切换动画（进入层次一）

用户按 ←（返回）
  → 当前步骤 > 0：reverse 上一步（元素退出）
  → 当前步骤 === 0：触发上一页切换
```

### 3.3 同一步内的并发 & 错开

同一 `data-step` 内的多个元素，支持设置 **错开时间（stagger）**：

```html
<!-- 同一步内，三个条目错开 0.1s 出现 -->
<ul data-step="2" data-stagger="0.1">
  <li data-animation="slide-right">条目 A</li>
  <li data-animation="slide-right">条目 B</li>
  <li data-animation="slide-right">条目 C</li>
</ul>
```

也支持给单个子元素各自设定独立 step（子元素级别细粒度控制）：

```html
<!-- 精细控制：每个 li 单独一步 -->
<li data-step="2" data-animation="slide-right">条目 A</li>
<li data-step="3" data-animation="slide-right">条目 B</li>
<li data-step="4" data-animation="slide-right">条目 C</li>
```

### 3.4 内置元素动画效果清单

| 效果名 | 描述 | 初始状态 → 最终状态 |
|--------|------|---------------------|
| `fade` | 淡入 | opacity:0 → 1 |
| `fade-up` | 从下方淡入 | opacity:0 + y:30px → 1 + y:0 |
| `fade-down` | 从上方淡入 | opacity:0 + y:-30px → 1 + y:0 |
| `fade-left` | 从右侧淡入 | opacity:0 + x:30px → 1 + x:0 |
| `fade-right` | 从左侧淡入 | opacity:0 + x:-30px → 1 + x:0 |
| `zoom-in` | 从小放大淡入 | opacity:0 + scale:0.8 → 1 + scale:1 |
| `zoom-out` | 从大缩小淡入 | opacity:0 + scale:1.2 → 1 + scale:1 |
| `slide-up` | 纯位移从下进入 | y:60px → y:0 |
| `slide-right` | 纯位移从左进入 | x:-60px → x:0 |
| `bounce` | 弹性进入 | scale:0 → scale:1（ease: back.out） |
| `typewriter` | 打字机效果 | 逐字符出现 |
| `none` | 无动画，直接显示 | visibility:hidden → visible |

## 4. 点击推进模型（核心机制）

### 4.1 状态机

每页 slide 维护一个 **SlidePlayState** 状态对象：

```typescript
interface SlidePlayState {
  /** 当前已 reveal 到哪一步（0 = 只有骨架，1 = 第一步已展示...） */
  currentStep: number
  /** 该页总共有多少个 reveal 步骤（不含 step=0 的骨架） */
  totalSteps:  number
  /** 当前是否有动画正在播放（防止连击） */
  isAnimating: boolean
}
```

### 4.2 点击推进逻辑（取代原来"直接切页"）

```typescript
function onAdvance() {
  const state = getCurrentSlidePlayState()

  if (state.isAnimating) return   // 动画播放中，忽略连击

  if (state.currentStep < state.totalSteps) {
    // 还有未展示的元素步骤 → 执行元素入场动画
    playStep(state.currentStep + 1)
  } else {
    // 本页所有步骤已展示完 → 执行页面切换
    navigateToNext()
  }
}
```

### 4.3 回退逻辑

```typescript
function onRetreat() {
  const state = getCurrentSlidePlayState()

  if (state.currentStep > 0) {
    // 回退一步：reverse 当前步骤的动画（元素退出）
    reverseStep(state.currentStep)
  } else {
    // 已是第 0 步 → 切换到上一页（上一页直接显示全部步骤：instant complete）
    navigateToPrev({ instantComplete: true })
  }
}
```

> **instantComplete**：切回上一页时，上一页所有 step 的元素立即显示为最终状态（不重放动画），这是符合 PPT 演讲直觉的行为。

### 4.4 进度指示器

底部 StatusBar 需要同时展示：
```
页码：3 / 16   步骤：2 / 4  ●●○○
```
小圆点代表当前页的动画步骤进度（类似 Keynote 的 build indicator）。

## 5. 技术方案选型

### 5.1 动画执行引擎

#### 方案 A：GSAP Timeline（推荐）

**理由**：
- 项目已全局引入 GSAP（`window.gsap`），无额外依赖
- `gsap.timeline()` 提供 `addPause()` / `addLabel()` 机制，天然适合"分步播放"
- 高性能（基于 RAF，GPU 加速），支持 CSS transform / opacity 等所有属性
- `tl.reverse()` 直接支持倒放（回退步骤）

**核心用法**：

```javascript
// 每页 slide 解析 data-step 属性后，构建一个 paused timeline
const tl = gsap.timeline({ paused: true })

// step 1：要点一淡入
tl.from('[data-step="1"]', { opacity: 0, y: 30, duration: 0.4, ease: 'power2.out' })
  .addPause()  // ← 暂停，等待下一次点击

// step 2：要点二弹入  
  .from('[data-step="2"]', { opacity: 0, scale: 0.8, duration: 0.35 })
  .addPause()

// step 3：总结飞入
  .from('[data-step="3"]', { opacity: 0, x: -40, duration: 0.3 })
  // 不加 addPause → 播放完自动视为"本页结束"

// 点击推进时：tl.resume()
// 回退时：tl.reverse()（支持原生倒放）
```

#### 方案 B：CSS Animation + class toggle

**优点**：无 JS，写 slide HTML 简单  
**缺点**：无法精确倒放（`.reverse()`），滑到之前步骤行为不自然；也无法做到 GSAP 那样精细的 ease 控制

#### 方案 C：Web Animations API（WAAPI）

**优点**：原生浏览器 API  
**缺点**：无 GSAP 的 timeline 概念，`getAnimations()` 管理复杂；兼容性比 GSAP 差

**结论**：选 **方案 A（GSAP Timeline）**，与项目现有技术栈完全一致。

---

### 5.2 动画配置的存储位置

**选项 1：直接写在 slide HTML 的 data 属性上**（推荐）

```html
<div class="slide" data-total-steps="3">
  <div class="point" data-step="1" data-animation="fade-up" data-duration="400">...</div>
</div>
```

优点：
- AI Agent 直接生成，无需额外配置文件
- 动画配置与 DOM 绑定，行号写回机制天然支持
- 人类可读，符合"每个 slide 是一个独立 HTML 文件"的设计哲学

**选项 2：JSON sidecar 文件**（`slide-003.animations.json`）

优点：配置与 HTML 分离  
缺点：增加文件数量，Agent 需要同时维护两个文件，复杂度高

**结论**：选**选项 1（data 属性）**，简单直接。

---

### 5.3 动画运行时的加载时机

```
useSlideRunner.navigateToInner()
  → DOM 注入完成
  → requestAnimationFrame（脚本执行后）
  → 🔥 新增：scanAndInitAnimations(host)
       解析 data-step 属性
       将 step≥1 的元素初始化为隐藏状态
       构建 GSAP Timeline（paused）
       注册到当前页的 SlidePlayState
```

## 6. 数据模型设计

### 6.1 Slide HTML 中的 data 属性规范

```
data-step="N"         页内步骤序号（0=立即显示，1/2/3...=逐步reveal）
data-animation="xxx"  元素入场动画效果（见 §3.4 清单）
data-duration="400"   动画时长，毫秒（默认 400）
data-delay="100"      额外延迟，毫秒（默认 0）
data-stagger="0.1"    子元素的错开时间，秒（仅对容器元素有效）
data-ease="power2.out" GSAP ease 字符串（默认 power2.out）

（Slide 根元素属性）
data-total-steps="N"  声明本页共有 N 个 reveal 步骤（可选，自动计算）
data-transition-in="fade"  本页入场切换效果（层次一）
data-transition-out="slide-left"  本页离场切换效果（层次一）
```

### 6.2 运行时状态（SlidePlayState）

```typescript
interface SlidePlayState {
  currentStep:  number    // 当前已显示到第几步
  totalSteps:   number    // 本页总步骤数（0 表示无步骤，直接切页）
  isAnimating:  boolean   // 防止连击
  timeline:     gsap.core.Timeline | null  // GSAP Timeline 实例（null = 无动画）
}
```

这个状态存在 `useSlideRunner` 或新建的 `useAnimationController` hook 中，
不进 React state（避免重渲染），直接用 `useRef` 持有。

## 7. 与现有架构的集成方式

### 7.1 改动点一览

| 文件 | 改动内容 | 优先级 |
|------|---------|--------|
| `useSlideRunner.ts` | 注入后调用 `initSlideAnimation()`，`navigateTo` 增加页面切换动画 | P0 |
| 新增 `useAnimationController.ts` | 封装 SlidePlayState + playStep / reverseStep 逻辑 | P0 |
| `StageArea.tsx` | `onAdvance` 由直接切页改为先查 SlidePlayState | P0 |
| `StatusBar.tsx` | 显示步骤进度点（●●○○） | P1 |
| `RightPanel/AnimateTab` | 编辑模式下配置元素动画（data-step / data-animation）| P2 |
| `packages/skills/agent/slide-generator/SKILL.md` | 补充动画 data 属性生成规范 | P1 |

### 7.2 关键接口

```typescript
// packages/viewer/src/hooks/useAnimationController.ts

export function useAnimationController(hostRef: RefObject<HTMLDivElement | null>) {
  const stateRef = useRef<SlidePlayState>({ ... })

  /**
   * 页面加载后调用：扫描 data-step 属性，构建 GSAP Timeline
   */
  function init(slideEl: HTMLElement): void

  /**
   * 点击推进：如果还有未 reveal 的步骤，播放下一步；否则返回 'done'
   * 返回值：'stepped'（播放了一个步骤）| 'done'（本页所有步骤完成，可切页）
   */
  function advance(): 'stepped' | 'done'

  /**
   * 回退：reverse 当前步骤；如果已是第 0 步，返回 'at-start'
   */
  function retreat(): 'retreated' | 'at-start'

  /**
   * 切换页面时的清理
   */
  function dispose(): void

  return { init, advance, retreat, stateRef }
}
```

### 7.3 点击推进流程改造（StageArea 伪代码）

```typescript
// 现在的逻辑
const onClickNext = () => runner.next()

// 改造后
const onClickNext = () => {
  const result = animCtrl.advance()
  if (result === 'done') {
    runner.next()   // 本页步骤全部完成，切页
  }
  // 'stepped' 情况：animCtrl 内部已经播放动画，不切页
}
```

## 8. 动画配置 UX 设计

### 8.1 总体原则

> **原则一：配置入口跟随选中状态**  
> 没有选中元素 → 配置"页面级切换动画"；选中元素 → 配置"元素入场动画"

> **原则二：所见即所得，预览优先**  
> 每个配置项改动后应提供"预览"按钮，让用户立刻看到效果，不需要进入演示模式

> **原则三：最小化学习成本**  
> 用户不需要懂 GSAP，不需要写代码，通过下拉选择 + 数值输入 + 步骤编号完成配置

---

### 8.2 层次一：页面切换动画的配置 UX

#### 入口 A：TopBar 全局切换效果

在 TopBar 编辑模式下，增加一个轻量下拉选择器：

```
[🎞 tang-slidex]  [演示] [编辑]  |  切换效果: [Fade ▾]  |  [保存] [放弃] [全屏]
                                              ↑
                              全局默认切换效果，影响所有页面
```

下拉选项：
- **无效果**（立即切换）
- **淡入淡出** Fade
- **向左滑动** Slide Left
- **向上推送** Slide Up
- **缩放进入** Zoom

选完后实时写入全局配置（`tang-slidex.config.json` 的 `defaultTransition`）。

#### 入口 B：RightPanel 未选中元素时 → "当前页"面板

当没有选中任何元素时，RightPanel 应从"空提示"变为"当前页设置"，显示：

```
┌─ 右侧面板 ────────────────────┐
│  当前页                        │
│  ─────────────────────────    │
│  入场切换                      │
│  [Fade ▾]                     │  ← data-transition-in
│                                │
│  离场切换                      │
│  [Slide Left ▾]               │  ← data-transition-out（可选，默认同入场）
│                                │
│  [▶ 预览切换效果]              │  ← 触发一次页面切换动画
│  ─────────────────────────    │
│  元素动画顺序                   │  ← 动画窗格（见 §8.3）
│  ...                          │
└────────────────────────────────┘
```

"预览切换效果"点击后：从当前页切到下一页再立刻切回（或仅展示入场动画），用户可以感受效果，不改变当前页码。

### 8.3 层次二：元素入场动画的配置 UX

分为两个子入口，相互联动：**动画窗格（AnimatePanel）** 和 **元素属性面板（AnimateTab）**。

---

#### 8.3.1 动画窗格（AnimatePanel）

**位置**：RightPanel 底部，始终可见（编辑模式下）。无论是否选中元素都展示。

```
┌─ 动画窗格 ─────────────────────────────────┐
│  本页动画顺序                                │
│                                             │
│  立即  🔷 [h2.slide-title] "三个核心优势"    │  ← step=0，点击选中该元素
│  ① 淡入↑ [div.card] "要点一"               │  ← step=1，拖拽可重排
│  ② 淡入↑ [div.card] "要点二"               │  ← step=2
│  ③ 弹入  [div.summary] "总结"              │  ← step=3
│                                             │
│  [+ 点击元素可添加动画]                      │
└─────────────────────────────────────────────┘
```

**核心交互**：
- 点击列表项 → 高亮选中舞台中对应的元素（双向联动：选中元素也会高亮对应列表项）
- 拖拽列表项 → 重排步骤顺序（更新 data-step 值）
- 点击步骤徽章（①②③）→ 切换选中，聚焦 AnimateTab 到该步骤
- 无 data-step 元素不出现在列表中

#### 8.3.2 AnimateTab（选中元素后的属性面板）

选中有 `data-step` 的元素后，RightPanel 的 Animate Tab 展示：

```
┌─ Animate Tab ────────────────────────────┐
│                                           │
│  动画步骤                                 │
│  ○ 立即显示（不参与步骤动画）              │  ← data-step="0"
│  ● 第  [1 ▲▼]  次点击后出现              │  ← data-step="1~N"
│                                           │
│  ─────────────────────────────────────   │
│                                           │
│  入场效果                                 │
│  ┌────────────────────┐                  │
│  │  淡入上升  fade-up  │  ← 预览缩略图    │
│  └────────────────────┘                  │
│  [< 切换效果 >]                           │
│                                           │
│  效果预设 ─────────────────────────────  │
│  [淡入] [淡入↑] [淡入↓] [淡入←] [淡入→]  │
│  [缩放入] [弹入] [打字机] [无动画]        │
│                                           │
│  ─────────────────────────────────────   │
│                                           │
│  时长     [400 ▲▼] ms                    │
│  延迟     [ 0  ▲▼] ms                    │
│  错开     [ 0.1▲▼] s  (子元素)           │  ← 仅当元素有多个子项时显示
│  缓动     [power2.out ▾]                 │
│                                           │
│  ─────────────────────────────────────   │
│                                           │
│  [▶ 预览此步骤动画]                       │  ← 从隐藏 → 播放入场 → 保持可见
│  [✕ 移除动画]                             │  ← 删除 data-step / data-animation
│                                           │
└────────────────────────────────────────────┘
```

**"无 data-step 元素"的 AnimateTab**：

如果选中的元素没有任何 `data-step` 属性：

```
┌─ Animate Tab ────────────────────────────┐
│                                           │
│  此元素无动画                              │
│  当前为立即显示（step=0）                  │
│                                           │
│  [+ 为此元素添加入场动画]                  │  ← 点击后自动分配 step=N+1
│                                           │
└────────────────────────────────────────────┘
```

---

#### 8.3.3 "预览此步骤动画"的行为

这是你提到的**编辑模式下触发一次让用户看效果**的功能：

```
用户点击 [▶ 预览此步骤动画]
  → 目标元素立即恢复到"动画开始前"的隐藏状态
     （不影响其他元素，其他 step 仍保持完全可见）
  → 播放该元素的入场动画（GSAP 单次 tween）
  → 动画结束，元素停留在最终可见状态
  → 不记录任何 patch（预览不影响保存状态）
```

---

### 8.4 编辑模式下的舞台视觉提示

编辑模式下，`data-step≥1` 的元素需要一些视觉提示，让用户知道"这个元素有动画步骤"：

```
方案 A：步骤徽章（推荐）
  元素的左上角叠加一个小徽章：
  [①] 表示 step=1，[②] 表示 step=2
  徽章仅在编辑模式下渲染（overlay 层），不影响实际 DOM

方案 B：选中框颜色区分
  有 data-step 的元素，SelectionBox 边框用橙色（区别于普通蓝色）
  配合 AnimatePanel 的列表高亮形成联动
```

两种方案可以组合：徽章 + 橙色边框。

### 8.5 UX 流程示意（完整配置流程）

```
演讲者想给"要点一"添加淡入动画，设为第1步
────────────────────────────────────────────
1. 进入编辑模式
2. 点击舞台中的"要点一"元素 → 选中（BLOCK_SELECTED）

```

## 9. AI Agent 如何生成动画描述

### 8.1 ppt-standards Skill 补充

Agent 在生成 slide HTML 时，遵循以下规则：

```
- 重要性递进的内容页，子要点必须用 data-step 分步 reveal
- 封面页（cover）通常无分步（data-total-steps="0"）
- 目录页（toc）无分步
- 内容页（content）建议 3-5 个 reveal 步骤
- 数据图表页可以先显示空图表骨架（step=0），再 reveal 数据（step=1）
```

### 8.2 典型生成示例（内容页）

```html
<!DOCTYPE html>
<html lang="zh">
<head>
  <style>
    /* 隐藏所有待 reveal 的元素（运行时 JS 会处理，这里只是保险） */
    [data-step]:not([data-step="0"]) { opacity: 0; }
  </style>
</head>
<body>
  <div class="slide" data-slide-type="content" data-transition-in="fade">

    <!-- step=0：立即显示 -->
    <h2 class="slide-title" data-step="0">三个核心优势</h2>
    <p class="subtitle" data-step="0">为什么选择 tang-slidex？</p>

    <!-- step=1：点击一次后 reveal，三个子项错开 0.1s -->
    <div class="card" data-step="1" data-animation="fade-up">
      <span class="card-icon">⚡</span>
      <h3>极速渲染</h3>
    </div>

    <!-- step=2 -->
    <div class="card" data-step="2" data-animation="fade-up">
      <span class="card-icon">🤖</span>
      <h3>AI 原生</h3>
    </div>

    <!-- step=3 -->
    <div class="card" data-step="3" data-animation="fade-up" data-delay="0">
      <span class="card-icon">✏️</span>
      <h3>可视化编辑</h3>
    </div>

  </div>
</body>
</html>
```

## 10. 已确认决策

| 问题 | 决策 | 说明 |
|------|------|------|
| **回退动画** | ✅ 原路动画退出 | `tl.reverse()`，与 Keynote 行为一致 |
| **编辑模式行为** | ✅ 立即显示所有步骤 + 支持单次预览 | 进入编辑模式跳过动画显示全部；AnimateTab 提供"预览此步骤动画"按钮 |
| **动画 Tab（RightPanel）** | ✅ 支持，P1 优先级 | AnimatePanel（动画窗格）+ AnimateTab（元素属性） |
| **Agent Skill** | 先写草稿 | 等基础能力稳定后再正式更新 ppt-standards Skill |
| **元素动画引擎** | GSAP Timeline | 项目已有，addPause() 分段推进 |
| **配置存储方式** | slide HTML 的 data-* 属性 | Agent 生成简单，行号写回天然支持 |
| **编辑模式视觉提示** | 步骤徽章 + SelectionBox 橙色边框 | 区分有/无动画元素 |
| **动画窗格位置** | AnimateTab 内部 | 只在切到 Animate Tab 时展示，节省面板空间 |
| **无 data-step 元素的默认行为** | 视为 step=0，立即显示 | 兼容 Agent 生成的旧 slide，不破坏已有内容 |

## 11. 待讨论与遗留问题

| 问题 | 说明 | 优先级 |
|------|------|--------|
| **GSAP Timeline 生命周期管理** | 换页时需要 `tl.kill()` 销毁旧 timeline，避免内存泄漏和动画污染 | 必须解决 |
| **与 slide 内 AI 生成动画的共存** | Agent 可能在 HTML `<script>` 里自己写了 GSAP 动画（如封面入场动画），需与框架 Build Timeline 区分，不互相干扰 | 约定：框架 timeline 只处理 `data-step` 元素；`<script>` 里的自定义动画是 step=0 的"立即播放"动画，不受步骤控制 |
| **data-step 写回源文件** | AnimateTab 改变 data-step 时，通过已有 `applyAttr()` 接口 → `AttrSetPatch` → slideSavePlugin 写回 | 已有接口，可复用 |
| **canvas/ECharts 隐藏兼容** | `data-step≥1` 初始隐藏时，canvas 尺寸可能为 0（`display:none`）导致 ECharts 渲染异常 | 用 `visibility:hidden + pointer-events:none` 代替 `display:none` |
| **切页时旧 slide DOM 的过渡保留** | 页面切换动画期间两页 DOM 需同时存在，当前 `innerHTML = ''` 直接清除需改造；可考虑双 slot 方案（outgoing/incoming 两个容器） | 开始实现层次一动画时解决 |
| **动画步骤编辑的拖拽重排** | AnimatePanel 的拖拽重排需更新所有相关元素的 data-step 值，涉及批量 AttrSetPatch | P2，可先用数字输入代替 |
