## 一、核心框架：用户视角 vs Agent 视角

tang-slidex 的动画系统服务于两类主体，需求**截然不同**，不强行统一：

| 维度 | 用户视角 | Agent 视角 |
|------|---------|-----------|
| **目标** | 可视化界面配置，无需写代码 | 最大化创意自由度，生成高质量效果 |
| **能力** | 不懂 GSAP，只需"选"而非"创作" | 精通前端，可写任意动画逻辑 |
| **存储偏好** | DOM data 属性（可读写回 HTML） | `<script>` 命令式 GSAP |
| **正交性** | 两者互不干扰，可共存 | |

## 二、四象限设计

```
               用户视角                      Agent 视角
             ┌────────────────────────┬──────────────────────────────┐
 页面级切换  │  AnimateTab 下拉选预设   │  data-transition-in 属性声明  │
 (Transition)│  写入 data 属性         │  （Phase 3 实现）              │
             ├────────────────────────┼──────────────────────────────┤
 元素级步骤  │  AnimateTab 配置面板     │  写法A: 纯声明式               │
 (Build)     │  data-step/animation   │  写法B: tang.onStep 自定义     │
             └────────────────────────┴──────────────────────────────┘
```

### Agent 标准写法（重构后全部 slide 遵守）

**写法 A：纯声明式**（推荐，支持可视化编辑）

```html
<div class="card" data-step="1" data-animation="fade-up">要点一</div>

<script>
  tang.onLoad(() => {
    gsap.from('.title', { y: -20, opacity: 0, duration: 0.45 })
  })
</script>
```

**写法 B：tang.onStep 自定义 GSAP**（高表现力，框架控制时机）

```html
<div class="card" data-step="1" data-animation="custom">要点一</div>

<script>
  tang.onLoad(() => {
    gsap.from('.title', { y: -20, opacity: 0, duration: 0.45 })
    // 粒子、canvas、hover 交互等自由写
  })
  tang.onStep(1, (els) => {
    // els = data-step="1" 的元素数组，框架传入
    gsap.from(els, { rotateY: 90, opacity: 0, duration: 0.6 })
  })
</script>
```

**迁移规则**：现有 `if (window.gsap) { ... }` 顶层写法，统一改为 `tang.onLoad(() => { ... })`。

## 三、window.tang 运行时 API

框架在 `<script>` 执行前注入 `window.tang`，收集注册但不立即执行：

```typescript
interface TangRuntime {
  onLoad(fn: () => void): void
  onStep(step: number, fn: (els: HTMLElement[]) => void): void
  onRetreat(step: number, fn: (els: HTMLElement[]) => void): void
  // 内部方法（框架调用）
  _runLoad(): void
  _getStepFn(step: number): ((els: HTMLElement[]) => void) | undefined
  _getRetreatFn(step: number): ((els: HTMLElement[]) => void) | undefined
}
```

## 四、交互与点击逻辑管理（核心 UX）

### 4.1 用户操作决策树

演示模式下，所有用户输入（点击 / 键盘 / 触摸）都经过同一个决策树：

```
用户输入（点击 / → / Space / 右滑）
         │
         ▼
    mode === 'present'？
         │ No → 忽略（编辑模式，由 EditManager 处理）
         │ Yes
         ▼
    isInteractiveTarget？（表格/链接/按钮等）
         │ Yes → 忽略（让原生事件处理）
         │ No
         ▼
    animCtrl.advance()
         │
    ┌────┴────┐
    │         │
 'stepped'  'done'
    │         │
    │    runner.next()（翻到下一页）
    │
    动画播放中，等待下次输入
```

```
用户输入（← / 左滑）
         │
         ▼
    mode === 'present'？
         │ No → 忽略
         │ Yes
         ▼
    animCtrl.retreat()
         │
    ┌────┴────┐
    │         │
'retreated'  'at-start'
    │         │
    │    runner.prev()（翻到上一页）
    │
    当前步骤元素瞬间隐藏
```

### 4.2 滚轮 / 点击 NavBar 按钮的特殊处理

- **NavBar `<` `>` 按钮**：**直接调用 `runner.prev()/next()`**，**绕过步骤动画**（让用户可以跳页）
- **侧边缩略图点击**：`runner.navigateTo(index)`，同样绕过步骤
- **滚轮**：暂不接入步骤动画，等同于 nav 按钮（直接翻页）

这符合 PPT 直觉：**键盘/点击推进步骤，按钮/缩略图直接跳页**。

### 4.3 NavigateTo 时的步骤重置

任何 `runner.navigateTo()` 调用（包括翻页触发的 next/prev）都会：
1. 换页（fetch + 注入）
2. `animCtrl.dispose()`：重置步骤状态到 0
3. `animCtrl.init(newSlide)`：扫描新页的 data-step

切回上一页时，上一页**从步骤 0 重新开始**（不恢复上次位置）。这是 v1 行为，符合简单性原则。

### 4.4 UI/UX 展示设计

**NavBar 步骤点（演示模式）**：

```
┌────────────────────────────────────────────────────────────────┐
│           [ < ]   1 / 16   [ ○ ● ○ ]   [ > ]   |   [ 编辑 ]  │
└────────────────────────────────────────────────────────────────┘
                              ↑ ↑ ↑
                   步骤点：已完成=实心紫色，未完成=空心
```

## 五、正确的时序（useSlideRunner 重构）

### 5.1 当前代码的 3 个 Bug

1. **CSS 动画与 GSAP 冲突**：设 `.slide { opacity:0, transition }` 后 script 里再 `gsap.from('.slide', ...)` 叠加
2. **`tang:slide-loaded` 过早触发**：在 rAF 之前派发，EditManager 绑定时 script 还没运行
3. **没有 `animCtrl.init()` 的正确插入点**：需要在 script 后、入场动画前

### 5.2 修正后的正确时序

```
navigateToInner(index, instant)：
  
  if !instant:
    host.style.opacity = '0'             ← 整个 host 淡出（不碰 .slide）
    host.style.transition = 'none'
  
  ① 注入 window.tang（重置上一页注册）
  ② 注入 <style> 节点
  ③ 注入 DOM 节点（不含 <script> 标签）
  ④ 提取所有 <script> 代码备用（不执行）
  
  requestAnimationFrame(() => {
    if !instant:
      host.style.transition = 'opacity 0.2s ease'
      host.style.opacity = '1'           ← host fade in
    
    ⑤ 逐个执行 <script>                  ← 收集 tang 注册（onLoad/onStep）
    
    ⑥ animCtrl.dispose()                 ← 清理上一页状态
    ⑦ animCtrl.init(slideEl)             ← 在 script 后！扫描 data-step，隐藏元素
    
    ⑧ tang._runLoad()                    ← 执行入场动画（此时 step 元素已隐藏）
    
    ⑨ rehydrateAll / highlight.js 等
    
    ⑩ dispatch 'tang:slide-loaded'       ← 最后！EditManager 重新绑定
  })
```

关键：
- **步骤 ⑤⑥⑦⑧ 都在同一个 rAF 里**，DOM 渲染只在这个 rAF 结束后才刷新
- **整个 host 做淡入淡出**，不碰 `.slide` 内部，与 GSAP 零干扰
- **`isAnimating` 锁只锁 `isNavigating`**（fetch/注入期间），`advance()/retreat()` 永远不被锁

### 5.3 移除 isAnimating 全局锁

当前 `isAnimating` 锁会在翻页动画期间拦截所有点击。这对步骤动画页面是错误的：
演示者在翻页动画期间点击应该**触发步骤推进**，而不是被丢弃。

修正：`isAnimating` 改为 `isNavigating`，仅锁住 `navigateTo` 的重复调用（防止同时 fetch 两个 slide），不影响 advance/retreat。

## 六、AnimationController 接口与实现

### 6.1 接口

```typescript
export interface AnimationController {
  init(slideEl: HTMLElement | null): void
  advance(): 'stepped' | 'done'
  retreat(): 'retreated' | 'at-start'
  revealAll(): void                       // 编辑模式
  previewStep(step: number): void         // AnimateTab 预览
  dispose(): void                         // 换页清理
  getStepMap(): Map<number, HTMLElement[]>
}
```

### 6.2 状态机（N 步需 N+1 次点击）

```
初始：currentStep=0, totalSteps=N

advance():
  currentStep >= totalSteps → 'done'（翻页信号，不播动画）
  otherwise:
    currentStep++
    els = stepMap.get(currentStep)
    playStepAnim(gsap, currentStep, els)  // 异步，不阻塞
    syncStore()
    return 'stepped'  // 最后一步也返回 'stepped'！

retreat():
  currentStep <= 0 → 'at-start'（翻到上一页信号）
  otherwise:
    els = stepMap.get(currentStep)
    gsap.killTweensOf(els)
    gsap.set(els, { opacity: 0 })
    els.forEach(el => el.style.visibility = 'hidden')
    currentStep--
    syncStore()
    return currentStep <= 0 ? 'at-start' : 'retreated'
```

### 6.3 playStepAnim()

```typescript
function playStepAnim(gsap, step, els) {
  // 优先检查 tang.onStep 自定义回调
  const customFn = window.tang?._getStepFn(step)
  if (customFn) {
    gsap.set(els, { visibility: 'visible' })
    customFn(els)
    return
  }
  // 使用预设
  const { animation, duration, delay, ease, stagger } = parseAnimAttrs(els[0])
  const preset = PRESETS[animation] ?? PRESETS['fade-up']
  gsap.set(els, { visibility: 'visible' })
  if (Object.keys(preset).length === 0) { gsap.set(els, { opacity: 1 }); return }
  gsap.from(els, { ...preset, duration, delay, ease,
    stagger: els.length > 1 ? (stagger || 0.08) : 0 })
}
```

### 6.4 init() 强制覆盖保底

```typescript
// 隐藏 step≥1 元素时，先 killTweensOf（防止 script 违规操作的残留 tween）
for each step 1..totalSteps:
  els = stepMap.get(step)
  gsap.killTweensOf(els)       // 杀掉可能存在的 tween
  gsap.set(els, { opacity: 0 })
  els.forEach(el => {
    el.style.visibility = 'hidden'
    el.style.opacity = '0'
    el.style.transform = ''    // 清除残留 transform
  })
```

## 七、数据属性规范

### 7.1 元素级

| 属性 | 默认值 | 说明 |
|------|--------|------|
| `data-step` | — | ≥1 参与分步，0 或不设立即显示 |
| `data-animation` | `fade-up` | 预设名或 `custom`（用 tang.onStep） |
| `data-duration` | `400`ms | 动画时长 |
| `data-delay` | `0`ms | 额外延迟 |
| `data-ease` | `power2.out` | GSAP ease |
| `data-stagger` | `0`s | 同组多元素错开时间 |

### 7.2 Slide 根元素

| 属性 | 默认值 | 说明 |
|------|--------|------|
| `data-transition-in` | `fade` | 页面切换效果（Phase 3） |

### 7.3 效果预设

| 效果名 | GSAP `from` |
|--------|-------------|
| `fade` | `{ opacity: 0 }` |
| `fade-up`（默认） | `{ opacity: 0, y: 30 }` |
| `fade-down` | `{ opacity: 0, y: -30 }` |
| `fade-left` | `{ opacity: 0, x: 30 }` |
| `fade-right` | `{ opacity: 0, x: -30 }` |
| `zoom-in` | `{ opacity: 0, scale: 0.85 }` |
| `zoom-out` | `{ opacity: 0, scale: 1.15 }` |
| `slide-up` | `{ y: 60 }` |
| `slide-right` | `{ x: -60 }` |
| `none` | `{}` |
| `custom` | 由 `tang.onStep(N, fn)` 提供 |

## 九、现有 slide 重构计划

所有 16 个 slide 按以下规则迁移：

**迁移规则 A**：`if (window.gsap) { ... }` → `tang.onLoad(() => { ... })`

```html
<!-- 之前 -->
<script>
  if (window.gsap) {
    gsap.from('.title', { y: -20, opacity: 0 })
  }
</script>

<!-- 之后 -->
<script>
  tang.onLoad(() => {
    gsap.from('.title', { y: -20, opacity: 0 })
  })
</script>
```

**迁移规则 B**：canvas/DOM 初始化代码不需要包在 tang.onLoad 里（它们不依赖 GSAP 时机），保持顶层执行即可：

```html
<script>
  // 这类代码不需要改，保持顶层执行
  const canvas = document.getElementById('particle-canvas')
  const ctx = canvas.getContext('2d')
  // ...粒子初始化...

  // 只有 GSAP 动画需要包进 tang.onLoad
  tang.onLoad(() => {
    gsap.from('.badge', { y: -24, opacity: 0 })
    gsap.from('.title', { y: 36, opacity: 0, delay: 0.15 })
  })
</script>
```

## 十、边界条件

| 场景 | 处理方式 |
|------|---------|
| 无 data-step 元素 | `totalSteps=0`，advance() 直接 `'done'`，点击翻页 |
| canvas 被 data-step 标记 | `visibility:hidden`（不用 display:none，canvas 尺寸不归零） |
| tang.onStep 和 data-step 不匹配 | `playStepAnim` 检查：有 custom 回调用，没有用预设 |
| 快速连点 | 不防抖，GSAP 自动处理覆盖 |
| 回退时动画正在播 | `gsap.killTweensOf` 立即终止 |
| 编辑→演示切换 | `setTimeout(0)` 等 DOM 还原后再 init |
| NavBar 按钮直接翻页 | 绕过 advance/retreat，直接 runner.next/prev |
| 缩略图点击跳页 | 绕过 advance/retreat，直接 runner.navigateTo |
